# 📖 Пошаговая инструкция: бот через GitHub Actions

**Это новая версия — без Railway. Всё делается на GitHub. Полностью бесплатно навсегда.**

Время на всё: **40-60 минут**.

---

## 🎯 ЧАСТЬ 1: Telegram-канал и бот

### Шаг 1.1. Создай канал

1. Telegram → меню → **Создать канал**
2. Название, описание — на твой вкус
3. Сделай **публичным**, придумай короткий адрес (например `@my_ai_news`)
4. Запомни этот адрес

### Шаг 1.2. Создай бота

1. В Telegram найди **@BotFather** (с синей галочкой!)
2. Напиши `/newbot`
3. Имя бота — любое (например `AI News Publisher`)
4. Username — должен заканчиваться на `bot` (например `my_ai_publisher_bot`)
5. BotFather пришлёт **токен** — длинная строка `1234567890:ABCdef...`
6. **СКОПИРУЙ И СОХРАНИ ТОКЕН!**

### Шаг 1.3. Добавь бота админом в канал

1. Канал → шестерёнка → **Администраторы** → **Добавить администратора**
2. Найди своего бота по username
3. Дай право **Публикация сообщений** (этого достаточно)

✅ **Готово.**

---

## 🧠 ЧАСТЬ 2: Получаем API-ключ Claude

### Шаг 2.1. Регистрация

1. Заходи на https://console.anthropic.com
2. Регистрация через Google или email
3. Подтверди email и телефон

### Шаг 2.2. Пополни баланс

1. Меню → **Plans & Billing** → **Add credit**
2. Пополни на **$5** (минимум, хватит на полгода+)
3. Нужна карта, работающая с зарубежными сервисами. Если своя не работает — поищи виртуальные карты для оплаты иностранных сервисов (Wildberries Pay, попроси кого-то из друзей за рубежом, или сервисы типа PayPalych).

### Шаг 2.3. Создай ключ

1. Меню → **API Keys** → **Create Key**
2. Назови как хочешь (например `tg-news-bot`)
3. **Скопируй ключ** (начинается с `sk-ant-api03-...`)
4. **СОХРАНИ В НАДЁЖНОЕ МЕСТО** — потом не покажет!

✅ **Готово.**

## 💻 ЧАСТЬ 3: GitHub — главный шаг

### Шаг 3.1. Регистрация на GitHub

1. https://github.com → **Sign up**
2. Создай аккаунт (бесплатно)
3. Подтверди email

### Шаг 3.2. Создай новый репозиторий

1. На главной нажми зелёную кнопку **New** (или плюсик в правом верхнем углу → New repository)
2. Имя репозитория: `tg-ai-news-bot` (или любое другое)
3. Выбери **Public** (важно для бесплатных GitHub Actions! Private тоже работает, но лимиты строже)
4. **Не отмечай** "Add a README file"
5. Нажми **Create repository**

### Шаг 3.3. Загрузи файлы

GitHub откроет страницу пустого репозитория. Среди ссылок найди **"uploading an existing file"** (она в центре).

1. Нажми на эту ссылку
2. **Перетащи в окно браузера ВСЕ файлы из папки `tg_ai_news_bot_v2`**, включая папку `.github`:
   - `bot.py`
   - `requirements.txt`
   - `posted_news.json`
   - `README.md`
   - `INSTRUCTIONS.md`
   - **Папка `.github`** (внутри неё `workflows/publish.yml` — это самое важное!)

⚠️ **Важно про папку `.github`:** GitHub в браузере может не сохранить структуру папок при перетаскивании. Если ты видишь, что после загрузки нет папки `.github/workflows/publish.yml` — действуй так:

1. На странице репозитория нажми **Add file** → **Create new file**
2. В поле имени файла напиши **точно так**: `.github/workflows/publish.yml`
   (GitHub автоматически создаст папки `.github` и `workflows`)
3. В большое окно скопируй содержимое из файла `publish.yml`, который у тебя на компе
4. Внизу — **Commit changes**

3. После того как все файлы загружены — внизу нажми **Commit changes**

### Шаг 3.4. Добавь секретные ключи в GitHub

Это самый важный шаг — здесь хранятся все твои пароли.

1. На странице репозитория сверху найди **Settings** (шестерёнка)
2. В левом меню → **Secrets and variables** → **Actions**
3. Нажми зелёную кнопку **New repository secret**
4. Добавь по очереди три секрета:

| Name | Secret |
|---|---|
| `TELEGRAM_BOT_TOKEN` | Токен от BotFather (шаг 1.2) |
| `TELEGRAM_CHANNEL_ID` | Адрес канала с `@`, например `@my_ai_news` |
| `ANTHROPIC_API_KEY` | Ключ Claude (шаг 2.3) |

Каждый секрет: вводишь имя → вставляешь значение → **Add secret**.

✅ **Готово.**

## ▶️ ЧАСТЬ 4: Первый запуск (проверка)

### Шаг 4.1. Запусти бота вручную

1. На странице репозитория наверху → вкладка **Actions**
2. Если GitHub попросит подтвердить — нажми **I understand my workflows, go ahead and enable them**
3. Слева увидишь **Publish AI News** — нажми
4. Справа кнопка **Run workflow** → **Run workflow** (зелёная)
5. Подожди 30-60 секунд → обнови страницу

### Шаг 4.2. Проверь результат

- Если кружок стал **зелёный ✅** — пост уже в твоём канале! Заходи смотри.
- Если **красный ❌** — нажми на запуск → нажми на job **publish** → раскрой шаг **Запустить бота** → посмотри, что красным цветом написано. Обычно это ошибка с секретами (опечатка в имени) или с ID канала.

### Самые частые ошибки и решения:

| Ошибка | Решение |
|---|---|
| `Chat not found` | Бот не добавлен в канал админом, или неправильный `TELEGRAM_CHANNEL_ID` (должен быть с `@`) |
| `Unauthorized` (от Telegram) | Неправильный `TELEGRAM_BOT_TOKEN` |
| `401` (от Anthropic) | Неправильный или истёкший `ANTHROPIC_API_KEY` |
| `insufficient credits` | Пополни баланс на console.anthropic.com |

✅ **Если первый пост опубликовался — всё работает. Дальше бот будет постить сам по расписанию: 9:00, 14:00, 19:00 по МСК.**

## 🎨 ЧАСТЬ 5: Настройка под себя (опционально)

### Поменять время публикаций

1. На GitHub открой файл `.github/workflows/publish.yml`
2. Нажми карандашик (Edit)
3. В блоке `schedule` поменяй cron. **Внимание: время в UTC, МСК = UTC + 3.**

Примеры:
```yaml
- cron: '0 6 * * *'   # 09:00 МСК
- cron: '30 8 * * *'  # 11:30 МСК
- cron: '0 13 * * *'  # 16:00 МСК
- cron: '0 18 * * *'  # 21:00 МСК
```

Формат: `минута час * * *`

4. Внизу — **Commit changes**

### Изменить число постов в день

В том же файле просто добавь или убери строки `- cron: ...`. Хочешь 1 пост — оставь одну строку. Хочешь 5 — добавь пять.

### Поменять источники новостей

1. Открой `bot.py` → карандашик → найди блок `RSS_SOURCES`
2. Добавь/убери источники
3. **Commit changes**

Хорошие RSS-ленты про ИИ:
- `https://openai.com/blog/rss.xml` — блог OpenAI
- `https://www.anthropic.com/news/rss.xml` — блог Anthropic
- `https://huggingface.co/blog/feed.xml` — Hugging Face
- `https://blog.google/technology/ai/rss/` — Google AI

### Изменить стиль постов

В `bot.py` найди переменную `SYSTEM_PROMPT` — это инструкция нейросети. Меняй под себя:
- Хочешь с юмором — добавь "Пиши с лёгким юмором"
- Короче — поменяй "Максимум 800 символов" на "Максимум 500"
- Своя подпись — добавь "В конце поста: 'Подписывайся → @твой_канал'"

После изменений GitHub Actions автоматически использует новую версию при следующем запуске.

## 🆘 Что делать если...

### Бот молчит несколько дней
1. GitHub → твой репозиторий → **Actions** → проверь, есть ли запуски
2. Если запусков нет — значит cron не работает. **Важно:** GitHub Actions для бесплатных репозиториев пауsит запуски, если в репозиторий 60 дней не было активности. Решение: просто сделай любую правку в любом файле и закоммить — это перезапустит расписание.
3. Если запуски есть, но красные — кликни на красный → посмотри логи

### Хочу запустить пост вручную прямо сейчас
- Actions → Publish AI News → Run workflow → Run workflow
- Через 30-60 секунд пост в канале

### Бот публикует мне в личку, а не в канал
- Значит `TELEGRAM_CHANNEL_ID` неправильный
- Должно быть `@my_channel_name` (с собачкой), а не имя бота

### Хочу видеть всё, что бот пишет в логе
- Actions → нажми на любой запуск → нажми на job **publish** → раскрой шаги. Там всё построчно.

### Хочу удалить пост, который уже опубликован — он не нравится
- Удалить можно в самом Telegram (зажми сообщение → удалить)
- Но бот его уже запомнил и **не повторит**. Если хочешь, чтобы повторил — в репозитории открой `posted_news.json` → удали последний ID → commit

### Хочу добавить картинки к постам
- RSS-ленты часто содержат картинку (поле `media_content`)
- Это усложнение — могу помочь, если попросишь

## 🎯 Что в итоге происходит

```
06:00 UTC (09:00 МСК) → GitHub запускает твой код
                     → Код парсит RSS → берёт неопубликованную новость
                     → Отправляет Claude → получает красивый русский текст
                     → Постит в твой канал
                     → Сохраняет ID в историю
                     → Завершается до следующего запуска

11:00 UTC (14:00 МСК) → То же самое со следующей новостью
16:00 UTC (19:00 МСК) → И ещё раз
```

И так каждый день, само, навсегда. 🚀
